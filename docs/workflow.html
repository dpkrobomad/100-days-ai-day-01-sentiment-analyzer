Sentiment API — workflow

Day 1 project: how traffic reaches VADER, and where FastAPI, Pydantic, and Streamlit sit.

1. System overview

Streamlit streamlit_app.py
Other clients curl · tests · webhooks
OpenAPI / Swagger browser @ /docs

From Streamlit — two modes

Path A default Python import → service.analyze_sentiment — no HTTP
⬇ skips API
service.py + VADER path A ends here; path B and external clients go through the API first

HTTP path (Streamlit toggle + other clients + Swagger)

Path B + external POST /api/v1/sentiment to Uvicorn
⬇
Uvicorn → FastAPI main.py · CORS · /health
⬇
Pydantic validates SentimentRequest / response shape
⬇
service.py + VADER same module as path A

One scoring implementation; the API adds validation, OpenAPI, and a stable wire format.

2. API request path

Client → POST JSON → FastAPI → Pydantic

invalid / empty text → 422

Pydantic OK → VADER in service → 200 JSON

On app startup, warm_vader() runs in lifespan so the first user request is not cold.

3. Streamlit — local engine only

User → Streamlit → analyze_sentiment → UI metrics + bar

Same return shape as the API; use model_dump(mode="json") to match wire format.
